# Archive

Historical material lives here so top-level `docs/` can stay focused on current product docs and assets.

- `incidents/` contains issue docs, plans, validation reports, and state files from past incident or environment-specific work.
- `rdw/` contains prior review-driven-workflow artifacts that are kept for audit history, not day-to-day editing.
- `strategy/` contains older positioning, research, and strategy notes that are no longer current-facing docs.

If a document is useful as historical context but should not shape the current product surface by default, move it here instead of keeping it at the top level.
